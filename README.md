# 1. PostgreSQL কী?
` PostgreSQL একটি শক্তিশালী, ওপেন সোর্স রিলেশনাল ডেটাবেজ ম্যানেজমেন্ট সিস্টেম (RDBMS) যা SQL (Structured Query Language) ও আরও অনেক ফিচার সমর্থন করে। এটি ACID (Atomicity, Consistency, Isolation, Durability) প্রিন্সিপল মেনে চলে, যার ফলে ডেটার নিরাপত্তা ও নির্ভরযোগ্যতা নিশ্চিত হয়।`

## PostgreSQL-এর মূল বৈশিষ্ট্যগুলোর মধ্যে রয়েছে:

` 1.কাস্টম ডেটা টাইপ তৈরি করার সুবিধা`

 `2.স্টোরড প্রসিডিউর ও ট্রিগার ব্যবহারের সুবিধা`

 `3.MVCC (Multi-Version Concurrency Control) সাপোর্ট`

`4.JSON ও XML ডেটার সাপোর্ট, অর্থাৎ স্ট্রাকচার্ড এবং আনস্ট্রাকচার্ড ডেটা উভয়ের জন্য কার্যকর`

 `উদাহরণ: ওয়াইল্ডলাইফ কনজারভেশন অ্যাপ্লিকেশনে PostgreSQL ব্যবহার করে rangers, species, এবং sightings টেবিল তৈরি করে সহজে তথ্য সংরক্ষণ ও বিশ্লেষণ করা যায়।`

# 2. PostgreSQL-এ ডেটাবেজ স্কিমার উদ্দেশ্য কী?
`ডেটাবেজ স্কিমা হলো ডেটাবেজের মধ্যে একটি লজিক্যাল গ্রুপিং যা বিভিন্ন টেবিল, ভিউ, ইনডেক্স, ফাংশন ইত্যাদি ধারণ করে। এক কথায়, স্কিমা হলো ডেটাবেজে অবজেক্ট সাজানোর একটি পদ্ধতি।`

## উদ্দেশ্য ও সুবিধা:

`1.একই ডেটাবেজে ভিন্ন ভিন্ন স্কিমা তৈরি করে একাধিক প্রজেক্ট পরিচালনা করা যায়।`

`2.ইউজার পারমিশন আলাদা করে দেওয়া যায় প্রতিটি স্কিমার জন্য।`

`3.কোড রিইউজ ও সংগঠনের জন্য স্কিমা অনেক কার্যকর।`

`উদাহরণ:  wildlife ডেটাবেজে public স্কিমার অধীনে species ও rangers টেবিল রাখতে পারি।`
`CREATE SCHEMA conservation;
CREATE TABLE conservation.sightings (...);`

# 3. Primary Key ও Foreign Key কী?
## Primary Key:
`Primary Key এমন একটি ফিল্ড বা কলাম যা প্রতিটি রেকর্ডকে ইউনিকভাবে শনাক্ত করে। এটি কখনোই NULL হতে পারে না এবং একাধিক রেকর্ডে এক ইন্ডেন্টিক্যাল ভ্যালু থাকতে পারে না।`

`উদাহরণ: ranger_id SERIAL PRIMARY KEY`

## Foreign Key:
`Foreign Key হলো এমন একটি কলাম যা অন্য একটি টেবিলের Primary Key এর সাথে সম্পর্ক স্থাপন করে। এটি ডেটার ইন্টিগ্রিটি নিশ্চিত করে।`

`উদাহরণ: ranger_id INTEGER REFERENCES rangers(ranger_id)`
`এখানে sightings টেবিলের ranger_id কলাম rangers টেবিলের ranger_id এর সাথে যুক্ত — এতে বুঝা যায়, এই sighting কোন রেঞ্জার করেছে ।`

# 4. VARCHAR এবং CHAR ডেটা টাইপের মধ্যে পার্থক্য কী?
## VARCHAR (Variable Character):

`এটি ভেরিয়েবল দৈর্ঘ্যের টেক্সট ডেটা সংরক্ষণ করে।`

`আমি যদি VARCHAR(20) দেই, এবং ৫টি অক্ষরের টেক্সট রাখি, তাহলে PostgreSQL শুধু ৫ অক্ষর সংরক্ষণ করবে।`

## CHAR (Character):

`এটি ফিক্সড দৈর্ঘ্যের স্ট্রিং সংরক্ষণ করে।`

`যদি CHAR(10) দেই এবং আমি মাত্র ৪ অক্ষরের শব্দ রাখি, তাহলে বাকি ৬টি জায়গায় স্পেস (padding) যোগ করে।`

`* পার্থক্য:`

` VARCHAR বেশি মেমোরি সাশ্রয়ী।`

 `CHAR পারফরম্যান্সের জন্য কিছু ক্ষেত্রে উপযোগী (যেমন: Country Codes — CHAR(2) = 'BD', 'US')।`


 #  5. SELECT স্টেটমেন্টে WHERE ক্লজের কাজ কী?
`WHERE ক্লজ SQL SELECT স্টেটমেন্টে ব্যবহার করা হয় রো বা ডেটা ফিল্টার করার জন্য। এটি এমন শর্ত নির্ধারণ করে যা পূরণ করলে তবেই ডেটা রিটার্ন হয়।`

`উদাহরণ: SELECT * FROM species WHERE conservation_status = 'Endangered';`
`এই কোয়েরিটি শুধুমাত্র সেই species দেখাবে যেগুলো Endangered হিসেবে চিহ্নিত।`

## WHERE ক্লজের মাধ্যমে আমি আরও নানা শর্ত দিতে পারি:

`1.তুলনা অপারেটর (=, >, <, !=)`

`2.লজিক্যাল অপারেটর (AND, OR, NOT)`

`3.প্যাটার্ন খোঁজার জন্য (LIKE, ILIKE)`

`4.রেঞ্জ খোঁজার জন্য (BETWEEN, IN)`
